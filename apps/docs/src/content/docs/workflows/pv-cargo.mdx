The **pv-cargo** group contains **{workflows.length} workflows** that target the physical `pv-cargo`
self-hosted runner connected via a Tailscale overlay network. These workflows handle deployments,
MCP topology inspection, the Experiment Engine, and runner maintenance.

## Notes

- **Discover / Inspect / Probe / Runner / Tailscale / Test** workflows have no `dispatch_inputs`
  and run with branch + environment defaults only.
- **Experiment Engine — Self-Hosted** is the most complex workflow — its `task` choice input drives
  which sub-command runs (smoke-test, lint, run-experiment, sync-supabase, collect-evidence).
- All pv-cargo workflows run on the `[self-hosted, pv-cargo]` runner label; they will fail (queue indefinitely)
  if the runner is offline.
