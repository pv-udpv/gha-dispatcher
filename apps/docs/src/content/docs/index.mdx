## What is GHA Dispatcher?

GHA Dispatcher is a single-page web application that gives you a focused, keyboard-friendly UI for triggering GitHub Actions `workflow_dispatch` events in the `pv-udpv/pplx-lab` repository.

Instead of navigating through the GitHub Actions UI (Actions tab → select workflow → Run workflow), you:

1. **Open the dispatcher** at its preview URL.
2. **Paste a fine-grained PAT** — the token is held only in React state (gone on tab close).
3. **Pick a workflow group** (pv-cargo / pv-sandbox / web) using the tab bar.
4. **Select the workflow** you want to trigger.
5. **Fill in dispatch inputs** (branch, environment, and any workflow-specific fields).
6. **Hit Dispatch** — the app calls `POST /api/dispatch` on the Express backend, which forwards to GitHub's API.

The recent-runs panel shows the last 20 runs so you can track status without leaving the page.

## Key design decisions

| Decision | Rationale |
|---|---|
| No auth backend | PAT is held in React state only — zero server-side token storage |
| Express proxy | Avoids CORS on GitHub API; logs dispatch events to Supabase |
| Bundled workflow catalog | `workflows.json` in `@gha-dispatcher/shared` avoids cold-start GitHub API calls |
| Monorepo (pnpm workspaces) | `apps/web`, `apps/docs`, `packages/shared` — consistent tooling |
| Starlight docs | Allows auto-generating catalog pages from the same `workflowsCatalog` used by the app |
